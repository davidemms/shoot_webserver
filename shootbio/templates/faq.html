SHOOT: sprout a new branch on the tree of life

FAQs

  1. How are the gene trees inferred?
    • Maximum likelihood tree inference from a multiple sequence alignment is used. The sequences for each database tree are aligned using MAFFT L-INS-i for up to 500 sequences or default MAFFT for larger alignments. Trees are inferred using FastTree with default parameters. This is performed by OrthoFinder, which also roots the tree using a rooted gene tree to identify the outgroup genes. A user's query gene is added to the alignment using MAFFT with the '--add' functionality and is added to the pre-computed gene tree using IQTREE's '-g' option. The user's tree is rooted on the same genes as the database tree.

  4. What databases does SHOOT use?
    • SHOOT will very soon offer a range of curated databases. The motivation for encouraging the use of curated databases over defaulting to the BLAST nr database is that these curated databases should make it easier to picture the evolution of the whole gene family rather than overwhelming the tree with thousands of near-identical hits. The current database is:
      • SHOOT UniProt Reference Proteomes 2020.

        • This database contains 78 species across all domains of life. UniProt says: "Reference proteomes have been selected among all proteomes (manually and algorithmically, according to a number of criteria) to provide broad coverage of the tree of life."

  5. Could my tree need to be manually re-rooted?
    • This is possible if your gene is an out-group to the species in the database you searched against. It's not possible to detect this programmatically so you should consider whether the tree should be re-rooted on your query gene. A far better approach is to search against a database which has a taxonomic range which covers your species. SHOOT should then be able to root the gene tree correctly. It isn't necessary for the database to actually contain your species.
