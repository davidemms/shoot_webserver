Help & Documentation

Options:

DIAMOND sensitivity

By default SHOOT performs a DIAMOND normal sensitivity followed by an "ultra-sensitive" search if that finds no hits. As an alternative, you can request that a single ultra-sensitive search instead immediately.

DIAMOND database

The search against a DIAMOND database is ued to assign a gene to the correct homology group prior to phylogenetic placement. The default database (profiles of homology groups) is made up of sequences for each group, selected using k-means clustering, that are most representative of that group. In most cases this works well and the correct homology group for a query gene is easily identified. The alternative option is to use the database containing all sequences from all homology groups, which may work better in some cases.

MAFFT Options

The alignment of the query sequence against the MSA of all the genes in its homology group is one of the most computationally expensive steps in a SHOOT query. Frr this reason MAFFT is used with options giving high performance: "--retree 1 --maxiterate 0 --nofft". You can instead chose for MAFFT to run using its default options, which is slower but more accurate.
